# v2.5.0 Release Announcement Draft

## Title
AnixOps Control Center v2.5.0

## Summary
AnixOps Control Center v2.5.0 is the first consolidated release in this line to ship validated realtime infrastructure, production-candidate AI/Vectorize workflows, and release-tested Web3/IPFS capabilities across Workers, Web, and Mobile.

This release closes the RC cycle with cross-surface validation, stronger automated coverage, Android integration execution on physical hardware, and a more consistent release baseline for commercial deployment and operational rollout.

---

## Highlights

### Realtime is now release-tested
- Workers SSE and WebSocket flows are covered by handler tests and end-to-end smoke tests.
- Web nodes/tasks realtime subscriptions were stabilized to prevent duplicate connections and duplicate event consumption.
- Mobile SSE reconnect and provider-side realtime updates were tightened and validated.

### AI and Vectorize are now part of the tested release path
- Workers AI generation, embeddings, log analysis, natural-language query translation, and assistant chat flows are covered by executable service tests.
- Vector insert/search/delete and anomaly helper paths are covered by service and handler tests.
- End-to-end tests validate embedding generation, vector insertion/search, assistant chat, and query translation through authenticated APIs.

### Web3 and IPFS moved from showcase to validated flow
- SIWE challenge and verify paths are covered in Workers tests.
- IPFS upload and retrieval are covered in Workers tests and E2E flow validation.
- On-chain audit persistence is validated through service tests and E2E coverage.
- Client payload formats were aligned with server verification requirements.

### Mobile release readiness improved
- Android integration tests were executed successfully on a connected Android 14 physical device.
- Android NDK configuration was updated to match current Flutter integration_test requirements.
- AI and Web3 entry points are reachable from mobile navigation.

---

## What changed

### Workers
- Added executable tests for SSE and WebSocket handlers.
- Added AI service and AI handler tests.
- Added Vectorize service and handler tests.
- Added Web3/IPFS service tests.
- Expanded E2E coverage for realtime, AI/Vectorize, and Web3/IPFS flows.

### Web
- Consolidated SSE connection handling into a safer shared lifecycle.
- Fixed route-transition cleanup behavior for nodes/tasks realtime subscriptions.
- Aligned Web3 verification client payload to include the signed message.

### Mobile
- Hardened SSE connection lifecycle and reconnect handling.
- Connected nodes/tasks providers to realtime events.
- Exposed AI and Web3 in the mobile shell navigation.
- Fixed Android integration build blockers in AI/Web3 UI code.

## Upgrade and rollout notes

- Review branch rollout order before production promotion.
- Ensure Cloudflare bindings for AI and Vectorize are present and correctly configured in deployment targets.
- Ensure Android toolchain and NDK alignment if reproducing mobile integration runs locally.
- For Web3 login flows, clients must submit `address`, `signature`, and `message` together for verification.

---

## Compatibility notes

- No intentional breaking API version bump is introduced in this release.
- This release does tighten some client/server expectations around realtime subscription behavior and Web3 verification payloads.
- Teams with custom clients should verify their Web3 verify request payload shape before production rollout.

---

## Known limitations

- Web3 wallet interaction on mobile still uses a simplified placeholder flow in the UI layer and should be expanded further in future releases.
- IPFS retrieval in the current Workers test environment validates stored/retrieved content through the R2-backed compatibility path rather than a full external gateway dependency.

---

## Commercial and operational readiness statement

This release is prepared using a staged RC-to-final promotion model, with prerelease gating before final release, cross-surface automated validation, and explicit release-prep verification of Workers, Web, and Mobile deliverables. Final production release should still follow environment-specific change management, approval, backup, and rollback procedures required by your organization.
